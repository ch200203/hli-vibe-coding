# 프롬프트 3 — 통합 테스트 (복붙용)

> Part-3 §7의 최종 통합 패턴 적용: 서버 실행 + 샘플 질문 5개 테스트

```
검수를 마쳤으니 통합 테스트를 돌릴 거야.

1단계: 서버 시작
- node server.js 실행
- CSV 11개 로드 완료 확인
- http://localhost:3000 접속 가능 확인

2단계: 샘플 질문 5개 테스트
각 질문을 POST /api/query로 보내고 결과를 확인해줘.

| # | 질문 | 예상 활성 리서처 | 예상 차트 | 검증 포인트 |
|---|------|----------------|----------|-----------|
| 1 | "30대 남성 고객이 가장 많이 가입한 상품 카테고리는?" | policy | bar | 카테고리별 건수 |
| 2 | "최근 12개월 손해율이 가장 악화된 보험사는?" | loss_ratio | line | 시계열 추이 |
| 3 | "적극형 고객의 평균 월 투자 가용금액은?" | investment | bar/none | 수치 정확 |
| 4 | "자녀 둘 이상 가구가 관심 있는 보장은?" | policy | bar/pie | child_count>=2 |
| 5 | "내일 주가가 오를까?" | 없음 (거절) | none | Guard 차단 |

3단계: 결과 정리
- 각 질문별 결과 요약 (성공/실패 + 실제 답변 첫 줄)
- 차트가 제대로 생성되었는지
- 거절 케이스가 올바르게 작동했는지
- 전체 통과율: N/5

4단계: 수정이 필요한 경우
- 실패한 질문의 원인 분석
- 해당 에이전트 코드 수정
- 다시 테스트

결과를 .omc/review/test_results.md에 저장해줘.
```
